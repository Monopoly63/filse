المرجع الدراسي

نظرة عامة وخطة المذاكرة

مرجع دراسي متكامل لمادة الذكاء الاصطناعي العملي، مبني حرفياً من محاضرات د. غادة صافي: شرح متدرّج من الأساسيات، أمثلة محلولة خطوة بخطوة، الأخطاء الشائعة، وامتحان تفاعلي يغطّي جميع أنماط الأسئلة المتوقّعة.

حول هذا المرجع
أُعدّ هذا المحتوى التفاعلي بواسطة Abdulmoin Hablas اعتماداً على محاضرات د. غادة صافي — هندسة المعلوماتية، السنة الثالثة. الهدف: مرجع مراجعة منظّم وتقييم ذاتي قبل الامتحان.
شو رح تتعلم؟
المادة بتتقسّم لـ 3 محاور كبيرة: (1) خوارزميات البحث (موجّه + غير موجّه) — وهي الأهم والأكثر بالامتحان، (2) تمثيل المعرفة والمنطق (قضايا + مسند)، (3) التعلّم تحت الإشراف (تصنيف + انحدار).

خريطة المادة

بايثون + البحث

المتغيرات، القوائم، القواميس، Queue/Stack، وبناء BFS/DFS من الصفر.

البحث غير الموجّه

BFS, DFS, UCS — التتبّع خطوة بخطوة، المسار، الأمثلية.

البحث الموجّه

Greedy, A*, Hill Climbing — الدوال الإرشادية h, g, f.

تمثيل المعرفة

أنواع المعرفة، منطق القضايا، جداول الحقيقة، التكافؤ المنطقي.

∀ منطق المسند

المسندات، المكمّمات، التوحيد، Modus Ponens/Tollens، الاستنتاج.

التعلّم تحت الإشراف

شجرة القرار، الانحدار الخطي، مقاييس التقييم.

خطة مذاكرة ليوم واحد قبل الامتحان

  1. الجولة الأولى (سريعة): اقرأ كل قسم مرة واحدة لفهم الفكرة العامة، بدون حفظ. ركّز على «ليش» مو بس «كيف».
  2. خوارزميات البحث: أهم شي. تأكد تعرف تتبّع كل خوارزمية يدوياً على رسم بياني، وتعرف الفرق بينها (متى أمثل؟ متى لأ؟).
  3. المنطق: احفظ القوانين (دي مورغان، contrapositive…) وافهم Modus Ponens/Tollens والتوحيد.
  4. التعلّم: افهم متى تصنيف ومتى انحدار + اقرأ الكود سطر سطر (الدكتورة بتجيب شرح كود).
  5. حُلّ الامتحان التفاعلي اللي بآخر الملف. كرّره حتى تجيب 100%. كل سؤال إله شرح حتى لو غلطت تتعلّم.
  6. راجع ورقة المراجعة السريعة آخر شي قبل ما تدخل القاعة.
نصيحة الدكتورة بتجيب 3 أنماط
حسب المحاضرات، الأسئلة بتجي بـ: تصحيح كود تتبّع خوارزمية شرح كود — بالإضافة لأسئلة نظرية (تصنيف، تكافؤ، استنتاج). الامتحان التفاعلي مغطّي كل هالأنماط.

أساسيات بايثون لبرمجة خوارزميات البحث

هاد القسم بيأسّس كل أدوات بايثون اللي رح تستخدمها ببرمجة BFS و DFS. كل بنية معطيات إلها دور محدّد بالبحث — افهم الدور مو بس الصيغة.

المتغيرات وأنواع البيانات

المتغير هو «صندوق» بنخزّن فيه قيمة. بالبحث بنمثّل فيه: اسم عقدة (نص)، تكلفة انتقال (رقم)، وهل العقدة تمت زيارتها (منطقي True/False).

node = "A"        # اسم عقدة (string)
cost = 5          # تكلفة انتقال (int)
visited = False   # هل تمت الزيارة؟ (bool)

print(node)
print(cost)
print(visited)

الجمل الشرطية — هل وصلنا للهدف؟

أساس أي خوارزمية بحث: نتحقّق إذا العقدة الحالية هي الهدف.

node = "G"
goal = "G"

if node == goal:
    print("وصلنا إلى الهدف")
ليش؟ كل خوارزمية بحث بتفحص هالشرط بكل خطوة. لو شالت عقدة وكانت = الهدف، توقف فوراً.

القوائم (List) — تمثيل الجبهة Frontier

القائمة بنية مرتّبة قابلة للتعديل. بنستخدمها للجبهة (العقد المنتظرة للتوسيع).

frontier = []          # قائمة فارغة
frontier.append("A")     # إضافة في النهاية
frontier.append("B")
frontier.append("C")
print(frontier)        # ['A','B','C']
frontier.pop()           # حذف آخر عنصر → ['A','B']
مهم
append() بتضيف بالنهاية، وpop() بتشيل من النهاية → هاد سلوك المكدّس Stack (LIFO) المستخدم بـ DFS.

أهم بنية بالمادة! بنمثّل فيها الرسم البياني: كل عقدة = مفتاح، وقيمته = قائمة جيرانها.

الرسم: A → B,C ، B → D ، C → E

graph = {
    "A": ["B", "C"],
    "B": ["D"],
    "C": ["E"],
    "D": [],
    "E": []

# الوصول لجيران A:
for neighbor in graph["A"]:
    print(neighbor)   # B ثم C

المجموعات (Set) — منع التكرار (visited)

المجموعة لا تسمح بالتكرار، والبحث فيها سريع جداً. بنستخدمها لتخزين العقد المزارة حتى ما ندور بحلقة لا نهائية.

visited = set()      # مجموعة فارغة
visited.add("A")
if "A" in visited:
    print("تمت زيارة العقدة")
ليش set مو list؟ فحص x in set سريع (زمن ثابت)، بينما x in list بطيء (يمرّ على كل العناصر). وبما إننا بنفحص الزيارة كتير، الـ set أكفأ.

    if node == goal:

الطابور Queue باستخدام deque

الطابور FIFO (الداخل أولاً يخرج أولاً) — قلب خوارزمية BFS. بنستخدم deque لأن الحذف من البداية فيها سريع.

from collections import deque
queue = deque()
queue.append("A")     # إضافة بالنهاية
queue.append("B")
queue.append("C")
queue.popleft()       # حذف من البداية (FIFO) → يشيل A

Stack مقابل Queue — الفرق اللي بحدّد DFS مقابل BFS

التمرين الشامل — BFS و DFS مع استرجاع المسار

هاد الكود الكامل من المحاضرة. بيستخدم قاموس parent لتذكّر «من وين أجينا لكل عقدة»، وبعدها نعيد بناء المسار للخلف.

# تمثيل الرسم البياني
graph = {
    "S": ["A", "B"],
    "A": ["C"],
    "B": ["D"],
    "C": ["G"],
    "D": ["G"],
    "G": []
}

# إعادة بناء المسار من البداية للهدف
def build_path(parent, goal):
    path = []
    current = goal
    while current is not None:
        path.append(current)
        current = parent[current]
    path.reverse()          # نعكسه ليبدأ من البداية
    return path

from collections import deque
def bfs(graph, start, goal):
    queue = deque([start])      # الجبهة = Queue
    visited = set()
    parent = {start: None}
    while queue:
        node = queue.popleft()  # FIFO
        if node == goal:
            return build_path(parent, goal)
        if node not in visited:
            visited.add(node)
            for neighbor in graph[node]:
                if neighbor not in parent:
                    parent[neighbor] = node
                queue.append(neighbor)

def dfs(graph, start, goal):
    stack = [start]             # الجبهة = Stack
    visited = set()
    parent = {start: None}
    while stack:
        node = stack.pop()      # LIFO
        if node == goal:
            return build_path(parent, goal)
        if node not in visited:
            visited.add(node)
            for neighbor in graph[node]:
                if neighbor not in parent:
                    parent[neighbor] = node
                stack.append(neighbor)

print("BFS:", bfs(graph, "S", "G"))
print("DFS:", dfs(graph, "S", "G"))
⚠ أخطاء شائعة بهالكود (الدكتورة بتجيبها بتصحيح الكود!)
  • استخدام queue.pop() بدل popleft() بالـ BFS → بيتحوّل لـ DFS!
  • نسيان visited → حلقة لا نهائية إذا في دورات بالرسم.
  • نسيان path.reverse() → المسار بيطلع معكوس (من الهدف للبداية).
  • استخدام list للـ graph بدل dictionary → ما بتقدر توصل للجيران بسهولة.
الخلاصة
الفرق الوحيد الجوهري بين BFS و DFS بالكود هو: BFS تستخدم Queue (popleft) وDFS تستخدم Stack (pop). كل الباقي متطابق.
الوحدة الثانية

خوارزميات البحث غير الموجّه Uninformed Search

«غير موجّه» يعني الخوارزمية ما عندها أي معلومة عن مكان الهدف — بتبحث بشكل أعمى حسب بنية الرسم فقط. الثلاثة هون: BFS وDFS وUCS.

المفاهيم الأساسية
  • العقدة (Node): حالة بفضاء البحث.
  • التوسيع (Expand): أخذ عقدة وإضافة كل جيرانها للجبهة.
  • الجبهة (Frontier): العقد المنتظرة للتوسيع.
  • المسار الأمثل: الأقصر (بعدد الخطوات) أو الأقل تكلفة.

① البحث بالاتساع أولاً — BFS (Breadth-First Search)

بتستكشف الرسم مستوى بعد مستوى: كل عقد العمق 1، بعدين كل عقد العمق 2… تستخدم طابور Queue.

✓ الميزة
تضمن إيجاد أقصر مسار (بعدد الخطوات) لأنها بتلاقي الهدف بأول مستوى يظهر فيه.
✗ العيب
تستهلك ذاكرة كبيرة (تخزّن كل المستوى الحالي).

مثال تتبّع كامل (من المحاضرة)

الرسم: S→A,B ، A→C,D ، B→E ، D→F,G ، والهدف G.

  1. نضع S بالطابور.
  2. نخرج S ونوسّعها → نضيف A, B.
  3. نخرج A → نضيف C, D.
  4. نخرج B → نضيف E.
  5. نخرج C (ليست الهدف، لا أبناء).
  6. نخرج D → نضيف F, G.
  7. نخرج E ، ثم F.
  8. نخرج G → الهدف! توقّف. ✓
ترتيب التوسيعS, A, B, C, D, E, F, G
المسار الناتجS → A → D → G
عدد العقد الموسّعة8 (مع G) أو 7 (قبل استخراج الهدف)
هل هو الأقصر؟نعم — لأن BFS تستكشف بالعمق التصاعدي فأول وصول = أقصر مسار

② البحث بالعمق أولاً — DFS (Depth-First Search)

بتغوص بفرع واحد لأقصى عمق، وإذا فشل بترجع للخلف (Backtracking) وتجرّب فرع تاني. تستخدم مكدّس Stack.

✓ الميزة
ذاكرة أقل (تخزّن فرع واحد فقط).
✗ العيب
لا تضمن أقصر مسار — ممكن تلاقي مسار طويل قبل القصير.

مثال تتبّع المكدّس خطوة بخطوة (من المحاضرة)

الرسم: S→A,B ، A→C ، C→F ، B→D,E ، E→G ، الهدف G.

انتبه! هون طلع المسار قصير لكن هاد مجرّد حظ بسبب بنية الشجرة. DFS عموماً لا تضمن أقصر مسار. لو كان G بفرع آخر أقصر، كانت DFS لاقت المسار الطويل أولاً.
شرح آلية الرجوع Backtracking لما DFS تسلك مسار S→A→C وتكتشف إنه ما بيوصل للهدف، بترجع لآخر نقطة فيها فرع غير مجرّب (وهي S)، وبتنتقل لـ S→B→D→G. هاد جوهر DFS: التعمّق أولاً، ثم الرجوع عند الفشل.

③ البحث ذو التكلفة المنتظمة — UCS (Uniform Cost Search)

لما يكون لكل حافة تكلفة، BFS ما بتكفي. UCS توسّع دائماً العقدة ذات أقل كلفة تراكمية من البداية، باستخدام طابور أولوية Priority Queue.

✓ مهم جداً
UCS تضمن المسار الأمثل (الأقل تكلفة) لأنها لا تتوقف إلا عند استخراج الهدف من طابور الأولوية بأقل كلفة ممكنة.

خوارزميات البحث الموجّه Informed / Heuristic Search

«موجّه» يعني عند الخوارزمية دالة إرشادية h(n) تقدّر «قد إيه العقدة قريبة من الهدف» — فتوجّه البحث بدل ما يكون أعمى. الثلاثة هون: Greedy وA* وHill Climbing.

المعادلات الأساسية — احفظها!
g(n) = التكلفة الفعلية من البداية حتى العقدة n
h(n) = التقدير الإرشادي للتكلفة من n إلى الهدف
f(n) = g(n) + h(n) = التقدير الكلّي للمسار عبر n
  • Greedy: نختار أصغر h(n) فقط.
  • A*: نختار أصغر f(n) = g(n)+h(n).
  • Hill Climbing: ننتقل للجار الأفضل محلياً فقط، بدون رجوع.

① البحث الجشع الأفضل-أولاً — Greedy Best-First Search

بكل خطوة نختار العقدة ذات أصغر h(n)، بدون النظر لتكلفة المسار اللي قطعناه. «جشع» لأنها بتاخد القرار اللي مبيّن أحسن دلوقتي.

مثال تتبّع (من المحاضرة)

القيم h: S(10) ، الأبناء A(7), B(4), C(6) ... الهدف G(0).

الخطوةالعقد المتاحة (h)المختارةالسبب
1A(7), B(4), C(6)Bh(B)=4 الأصغر
2F(3)Fالابن الوحيد لـ B
3J(1)Jالابن الوحيد لـ F
4G(0)Gوصلنا للهدف ✓

المسار: S → B → F → J → G

✗ تحذير مهم
Greedy لا تضمن المسار الأقل تكلفة! لأنها تتجاهل g(n) تماماً وتنخدع بالقرب الظاهري من الهدف.

② خوارزمية A* (A-Star)

الأقوى. تجمع بين التكلفة الفعلية g(n) والتقدير الإرشادي h(n) → f(n)=g+h. بكل خطوة نختار أصغر f.

مثال تتبّع كامل مع تحديث المسار (من المحاضرة)

البداية A. التكاليف: A→B=1, A→C=4, C→D=1, D→Goal=2, B→D=3. وقيم h معطاة.

الخطوةالعقدةghf=g+hOPEN بعدها
1: نوسّع AB156[B(6), C(6)]
C426
2: نوسّع CD (عبر C)516[D(6), B(6)]
3: نوسّع DGoal707[B(6), Goal(7)]
4: نوسّع BD (عبر B)415[D(5), Goal(7)]
5: نوسّع D ثانيةًGoal606[Goal(6)] ✓

المسار النهائي: A → B → D → Goal   الكلفة: 1+3+2 = 6

ليش A* اختارت هذا المسار مو A→C→D→Goal؟ لأن كلفة A→C→D→Goal = 4+1+2 = 7، بينما A→B→D→Goal = 6. A* تكتشف المسار الأرخص عبر تحديث g(D) من 5 إلى 4 لما توصله عبر B. هون نقطة قوة A*: تأخذ التكلفة الفعلية بالحسبان، عكس Greedy.
كسر التعادل (Tie-breaking)
لما عندك عقدتين بنفس f، اختار اللي عندها h أصغر (أقرب للهدف). إذا تساوت h كمان، حسب ترتيب الإدراج.

مفهوم المقبولية Admissibility

الدالة الإرشادية مقبولة إذا حقّقت الشرط: h(n) ≤ h*(n) أي لا تبالغ أبداً بتقدير التكلفة الحقيقية (h* = التكلفة الحقيقية).

A79مقبولة (7≤9)

⚠ أثر عدم المقبولية
إذا الدالة غير مقبولة (بتبالغ)، A* قد لا تضمن الحل الأمثل وممكن تبتعد عن مسار جيد بسبب تقدير مبالغ فيه.

③ تسلّق التل — Hill Climbing

خوارزمية بحث محلي: من الحالة الحالية، تنتقل للجار الأفضل فقط إذا كان أحسن منها. لا رجوع للخلف. تتوقف لما ما يبقى جار أفضل.

الهدف
عادةً نبحث عن أعلى قيمة (القيمة تمثّل جودة الحالة). ننتقل للجار الأعلى دائماً.

مثال تتبّع (من المحاضرة)

البداية A(2). الجيران B(5), C(4). القيمة = الجودة (نبحث عن الأعلى).

الحالة الحاليةالجيرانالاختيار
A(2)B(5), C(4)B(5) — الأفضل وأعلى من 2
B(5)D(7)D(7) — أعلى من 5
D(7)E(6)توقّف! E(6)<7 لا تحسين

المسار: A → B → D ، توقفت عند D(7).

✗ المشاكل الثلاث الكبيرة لـ Hill Climbing
  • الأقصى المحلي (Local Maximum): تتوقف عند قمة محلية أفضل من جيرانها لكن مو الأفضل عالمياً. مثال: توقفت عند F(9) رغم وجود L(10) بفرع آخر لم تستكشفه.
  • الهضبة (Plateau): كل الجيران بنفس القيمة (مثلاً كلهم 6) → ما في اتجاه واضح للتحسّن فتتوقف.
  • الحافة/القمة الضيقة (Ridge): صعوبة الصعود لأن الاتجاهات المتاحة كلها تنزل.
المعالجات
Random Restart Hill Climbing Simulated Annealing السماح بحركات جانبية محدودة توسيع مفهوم الجوار

الفرق الجوهري: Greedy مقابل A* (أهم مقارنة بالقسم!)

Greedy
تعتمد على h(n) فقط. سريعة لكن قد تعطي مسار غير أمثل لأنها تتجاهل التكلفة المقطوعة.

مثال: المسار S→B→E→G بكلفة 13.
A*
تعتمد على f=g+h. تجمع التكلفة الفعلية مع التقدير → تعطي المسار الأمثل (إذا h مقبولة).

نفس المثال: S→A→D→G بكلفة 10 ✓
الخلاصة
A* أفضل من Greedy لأنها لا تنخدع بالقرب الظاهري من الهدف — تأخذ التكلفة المقطوعة (g) بعين الاعتبار أيضاً.

منطق القضايا (Propositional Logic)

القضية = عبارة تكون إما صادقة (T) أو كاذبة (F). نربط القضايا برموز منطقية.

الرمزالاسمالمعنى
¬Pالنفي NOTليس P
P ∧ Qالعطف ANDP و Q معاً
P ∨ Qالفصل ORP أو Q
P → Qالشرط IFإذا P فإن Q
P ↔ Qالتكافؤ IFFP إذا وفقط إذا Q

أمثلة تحويل جُمل لصيغ منطقية

الجملةالصيغة
إذا كان مجتهداً وحضر فإنه ينجح(D ∧ A) → P
الطالب لا ينجح إذا لم يدرس¬S → ¬P
إذا لم يتصل بالإنترنت فلن يحدّث القاعدة¬I → ¬U
يعطي إنذاراً إذا الحرارة مرتفعة أو وُجد دخان(H ∨ M) → W

الاستنتاج بسلسلة قواعد (مثال أحمد)

القواعد: R→S ، Attend80→A ، (S∧A)→E
المعطيات: R ، Attend80

الاستنتاج: من R→S و R نستنتج S ← من Attend80→A نستنتج A ← من (S∧A)→E نستنتج E → يحق لأحمد دخول الامتحان ✓

تصنيف القضايا: توتولوجيا / تناقض / صحة عرضية

القضيةالتصنيفالسبب
(P∧Q) → Qتوتولوجيادائماً صادقة
(P∨Q) ∧ ¬Pصحة عرضيةتصدق أحياناً (P=F,Q=T)
(P→Q) ∧ P ∧ ¬Qتناقضدائماً كاذبة (مستحيل)
(P↔Q) → (Q↔P)توتولوجياالتكافؤ تبادلي
(P∨Q) → Qصحة عرضيةتكذب عند P=T,Q=F
تعريفات
  • توتولوجيا (Tautology): صادقة في كل الحالات.
  • تناقض (Contradiction): كاذبة في كل الحالات.
  • صحة عرضية (Contingency): صادقة في بعض الحالات وكاذبة في أخرى.

قوانين التكافؤ المنطقي (تُثبَت بجداول الحقيقة)

القانونالصيغة
دي مورغان الأول¬(P ∧ Q) ≡ (¬P ∨ ¬Q)
دي مورغان الثاني¬(P ∨ Q) ≡ (¬P ∧ ¬Q)
تحويل الشرط لفصل(P → Q) ≡ (¬P ∨ Q)
ضد الموجب Contrapositive(P → Q) ≡ (¬Q → ¬P)
نفي الشرط¬(P → Q) ≡ (P ∧ ¬Q)
جدول حقيقة: إثبات دي مورغان ¬(P∧Q) ≡ (¬P∨¬Q)
PQP∧Q¬(P∧Q)¬P¬Q¬P∨¬Q

العمودان ¬(P∧Q) و ¬P∨¬Q متطابقان → التكافؤ ثابت. مثال واقعي: «ليس صحيحاً أن الجو حار وممطر معاً» ≡ «إما الجو ليس حاراً أو ليس ممطراً».
جدول حقيقة: نفي الشرط ¬(P→Q) ≡ (P∧¬Q)

⚠ محدودية منطق القضايا
«كل طالب في المعلوماتية يدرس البرمجة» — بمنطق القضايا لازم نكتب قاعدة لكل طالب (A→PA, L→PL, K→PK...). لو ألف طالب = ألف قاعدة! الحل: منطق المسند يكتبها مرة واحدة بمتغير: ∀x (InfoStudent(x) → StudiesProgramming(x))

منطق المسند ومنطق الدرجة الأولى First-Order Logic

منطق المسند (First-Order Logic) أقوى من منطق القضايا: بيستخدم مسندات (Predicates) ومتغيرات ومكمّمات (Quantifiers) فبيقدر يعبّر عن «كل» و«يوجد» بصيغة واحدة معمّمة.

المسندات حسب الرتبة (عدد الوسطاء)

المسندالرتبةالقراءة
Student(Mona)أحادي (1 وسيط)منى طالبة
Teaches(Nader, Logic)ثنائي (2)نادر يدرّس المنطق
Gives(Ali, Book, Sara)ثلاثي (3)علي يعطي كتاباً لسارة
Between(C, A, B)ثلاثي (3)C يقع بين A و B

المكمّمات (Quantifiers)

∀ المكمّم الكلّي (For All)
∀x P(x) = «لكل x، P صحيحة». يُستخدم مع → غالباً.
∃ المكمّم الوجودي (Exists)
∃x P(x) = «يوجد x واحد على الأقل، P صحيحة». يُستخدم مع ∧ غالباً.

أمثلة صياغة

«كل طالب مسجّل بالجامعة يدرس مقرراً واحداً على الأقل»:

∀x ((Student(x) ∧ Registered(x,Univ)) → ∃y (Course(y) ∧ Studies(x,y)))

«يوجد طالب لا يحب الرياضيات ولكنه يحب البرمجة»:

∃x (Student(x) ∧ ¬Likes(x,Math) ∧ Likes(x,Programming))
⚠ ترتيب المكمّمات يغيّر المعنى!
  • ∀x ∃y Advises(y,x) = لكل شخص x يوجد ناصح y (قد يختلف الناصح).
  • ∃y ∀x Advises(y,x) = يوجد شخص واحد y ينصح الجميع.
الصيغتان غير متكافئتين! تبديل ∀ و∃ يغيّر المعنى.
✗ خطأ شائع بالمكمّمات (الدكتورة بتجيبه بتصحيح!)
«كل طبيب يعمل في مستشفى» — الصيغة الخاطئة: ∀x Doctor(x) ∧ WorksIn(x,Hospital)
تعني أن كل شيء طبيب ويعمل بمستشفى! الصحيح استخدام →:
∀x (Doctor(x) → WorksIn(x,Hospital))   أو الأدق:
∀x (Doctor(x) → ∃y(Hospital(y) ∧ WorksIn(x,y)))
القاعدة: مع ∀ استخدم → (شرط)، ومع ∃ استخدم ∧ (عطف).

قواعد الاستنتاج (Inference Rules)

Modus Ponens (إثبات بالإيجاب)

P → Q ,  P  ∴  Q

مثال: Researcher(Nora)→WritesPaper(Nora) و Researcher(Nora) ∴ WritesPaper(Nora).

Modus Tollens (إثبات بالنفي)

P → Q ,  ¬Q  ∴  ¬P

مثال: Verified(File7)→Accepted(File7) و ¬Accepted(File7) ∴ ¬Verified(File7).

التوحيد (Unification)

إيجاد تعويض يجعل عبارتين متطابقتين. مهم جداً للاستنتاج بـ FOL.

✓ توحيد ناجح
Supervises(x, ProjectA) مع Supervises(DrHadi, y)
التعويض: {x/DrHadi, y/ProjectA}
الناتج: Supervises(DrHadi, ProjectA)
✗ توحيد فاشل
Parent(x, x) مع Parent(Ali, Sara)
يتطلب x=Ali و x=Sara معاً، لكن Ali≠Sara → التوحيد فاشل.

الاستنتاج الأمامي مقابل الخلفي

⏩ الأمامي (Forward Chaining)

نبدأ من الحقائق ونطبّق القواعد لنستخرج كل النتائج الممكنة.

مثال: Student(Huda) ، ∀x Student(x)→HasID(x) ، ∀x HasID(x)→CanBorrow(x)
← HasID(Huda) ← CanBorrowBooks(Huda) ✓

⏪ الخلفي (Backward Chaining)

نبدأ من الهدف ونرجع للوراء نبحث عن قواعد تثبته.

مثال: الهدف CanGraduate(Samir) ← نحتاج CompletedCourses(Samir) ← نحتاج PassedAllExams(Samir) (موجود حقيقة) → الهدف يثبت ✓
⚠ فشل الإثبات بسبب نقص حقيقة
الهدف ReceivesCertificate(Maya) يحتاج AttendedWorkshop(Maya) ∧ PassedTest(Maya).
الأول موجود، لكن PassedTest(Maya) غير موجود → لا يمكن إثبات الهدف لأن قاعدة المعرفة ناقصة.

مثال استنتاج متعدّد المسارات

المعطيات: Researcher(Nabil) ، Publishes(Nabil)
القواعد: Researcher(x)→Academic(x) ، Publishes(x)→Active(x) ، (Academic(x)∧Active(x))→Eligible(x)

الاستنتاج: Academic(Nabil) + Active(Nabil) ← EligibleForGrant(Nabil) ✓
✗ حدود منطق الدرجة الأولى التقليدي
«غالباً الطلاب المجتهدون ينجحون» — يصعب تمثيلها بدقة لأن «غالباً» تعبّر عن معرفة احتمالية وليست حكماً قطعياً (صادق/كاذب). نحتاج: المنطق الاحتمالي، المنطق الضبابي (Fuzzy)، أو النماذج الإحصائية.

① التصنيف بشجرة القرار (Decision Tree)

سلسلة أسئلة شرطية تقسّم البيانات تدريجياً حتى نصل لفئة نهائية. ميزتها: قريبة من المنطق الشرطي وسهلة الرسم والشرح.

خطوات التطبيق العملي

  1. تجهيز البيانات في مصفوفة خصائص X وقائمة فئات y.
  2. تحديد الخصائص (Features) — مثل ساعات الدراسة ونسبة الحضور.
  3. تحديد الفئة الهدف (Label) — مثل ناجح/راسب.
  4. تدريب النموذج DecisionTreeClassifier.
  5. استخدام predict لتصنيف طالب جديد.
  6. تقييم النموذج باستخدام Accuracy أو مقاييس أخرى.

الكود الكامل (من المحاضرة)

from sklearn.tree import DecisionTreeClassifier, plot_tree
import matplotlib.pyplot as plt

# بيانات التدريب: [ساعات الدراسة, نسبة الحضور]
X = [[5,90],[4,85],[2,40],[1,35],
     [6,95],[3,60],[4,75],[2,50]]
# النتائج الصحيحة (الفئات)
y = ["ناجح","ناجح","راسب","راسب",
     "ناجح","راسب","ناجح","راسب"]

model = DecisionTreeClassifier(random_state=0)
model.fit(X, y)            # تدريب النموذج

new_student = [[5, 80]]
prediction = model.predict(new_student)
print("النتيجة المتوقعة:", prediction[0])

② الانحدار الخطي (Linear Regression)

التنبّؤ بقيمة رقمية مستمرة (مثل درجة طالب أو سعر منزل). يحاول إيجاد خط مستقيم يصف العلاقة بين المتغير المستقل X والتابع y.

y = a·x + b    (a = الميل slope ، b = الثابت/التقاطع intercept)

الكود الكامل (من المحاضرة)

from sklearn.linear_model import LinearRegression
from sklearn.metrics import mean_squared_error, r2_score
import numpy as np

X = np.array([[1],[2],[3],[4],[5],[6]])  # ساعات الدراسة
y = np.array([50,60,68,80,88,95])     # الدرجات

model = LinearRegression()
model.fit(X, y)

prediction = model.predict(np.array([[7]]))
print("الدرجة المتوقعة:", prediction[0])
print("الميل a:", model.coef_[0])
print("الثابت b:", model.intercept_)

y_pred = model.predict(X)
print("MSE:", mean_squared_error(y, y_pred))
print("R2:", r2_score(y, y_pred))

  • BFS أمثل بالخطوات، UCS و A* أمثل بالتكلفة، DFS و Greedy و Hill Climbing مو أمثل.

  • Hill Climbing تعلق بالأقصى المحلي والهضبة.
